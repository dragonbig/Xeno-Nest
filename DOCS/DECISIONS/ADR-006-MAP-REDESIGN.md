# ADR-006: 맵 구조 전면 재설계 — 내부 기지 + 외부 행진 공간

> **카테고리:** DECISIONS
> **최초 작성:** 2026-03-23
> **최종 갱신:** 2026-03-23
> **관련 기능:** 맵 생성, 적 이동, 스폰 시스템, 카메라

**상태:** Accepted
**날짜:** 2026-03-23

---

## 맥락

기존 맵(12×16, 총 192타일)은 외벽이 직사각형으로 맵 전체를 감싸고, 5개 입구가 외벽 가장자리에 시계 방향으로 분산 배치된 구조였다. 적은 입구에서 스폰하자마자 즉시 기지 내부에 진입하여, 스폰과 전투 사이의 거리가 매우 짧았다. 이로 인해 두 가지 문제가 있었다:

1. **행진 연출 부재:** 적이 스폰 직후 바로 성벽에 도달하여, 대규모 적이 몰려오는 시각적 긴장감이 없었다.
2. **입구 분산으로 인한 방어 난이도:** 5개 방향에서 동시에 적이 진입하므로 모든 방향을 방어해야 했고, 타워 배치의 전략적 집중이 어려웠다.

---

## 결정

### 1. 맵 크기 확대: 12×16 → 30×24

전체 맵을 30열 × 24행(총 720타일)으로 확대한다. 기지 외부에 넓은 행진 공간을 확보하여 적이 스폰 지점에서 기지 입구까지 긴 거리를 이동하도록 한다.

### 2. 불규칙 타원형 외벽으로 기지 감싸기

기존의 직사각형 외벽 + 4개 모서리 방식을 폐기하고, 파괴 불가능한 BLOCKED 타일로 이루어진 불규칙 타원 형태의 외벽으로 기지를 감싼다. 기지 내부의 건설 가능 공간은 ~125타일로 기존과 유사한 규모를 유지한다.

### 3. 기지 입구 1개로 통일 (2타일 폭)

기존 5개 입구를 1개로 통합한다. 입구는 row 13, col 14~15에 위치하며 2타일 폭이다. WALL 건물 2개를 배치하면 입구를 완전히 봉쇄할 수 있다.

### 4. 스폰 지점을 맵 하단 가장자리로 이동

5개 스폰 지점을 모두 맵 하단(row 22)에 균등 배치한다(col 5, 10, 15, 20, 25). 적은 하단에서 스폰하여 기지 입구까지 약 9행(432px) 이상을 행진한다.

### 5. 적 외벽 통과 방지 로직 추가

직선 추적(Direct Pursuit) AI가 외벽을 관통하지 못하도록 두 가지 보호 로직을 추가한다:
- **pursueTarget 슬라이딩:** 이동 후 BLOCKED 감지 시 전체 롤백 → X축만 시도 → Y축만 시도 (3단계)
- **separation 롤백:** 충돌 해소 push 후 BLOCKED 진입 시 push를 롤백

## 이유

### 행진 연출이 게임 경험의 핵심이다

타워 디펜스 장르에서 적이 멀리서 몰려오는 모습은 긴장감과 전략적 판단의 시간을 제공한다. 기존 맵은 스폰과 전투가 거의 동시에 발생하여 이 경험이 부재했다. 외부 행진 공간을 통해 적의 접근을 시각적으로 확인하고 방어를 준비할 시간을 확보한다.

### 입구 1개는 방어 전략의 깊이를 높인다

5개 입구는 방어 자원을 분산시켜 전략적 선택의 여지가 적었다. 입구 1개로 통합하면 입구 방어에 자원을 집중할지, 기지 내부에 2차 방어선을 구축할지 등 전략적 깊이가 생긴다. 2타일 폭이므로 WALL 2개로 봉쇄 가능하며, 봉쇄 시 적이 벽을 공격해야만 진입할 수 있다.

### 불규칙 외벽은 자연스러운 기지 형태를 제공한다

직사각형 외벽은 인공적으로 보였다. 불규칙 타원 형태는 둥지(NEST)라는 게임 세계관에 부합하며, 내부 건설 공간의 형태도 다양해져 배치 전략에 변화를 준다.

## 검토한 대안

### 대안 A: 기존 맵 크기 유지 + 입구 수 감소

- 12×16 맵에서 입구만 1~2개로 줄이는 방식.
- 장점: 맵 크기 변경 없이 방어 전략 단순화 가능.
- 기각 이유: 행진 공간이 없으므로 핵심 목표인 행진 연출을 달성할 수 없다.

### 대안 B: 맵 확대 + 입구 3개 유지

- 30×24 맵에 입구를 3개(좌, 중앙, 우)로 배치.
- 장점: 방어 포인트가 여전히 분산되어 난이도 유지.
- 기각 이유: 불규칙 타원형 외벽에 3개 입구를 자연스럽게 배치하기 어렵고, 1개 입구로 충분한 전략적 깊이가 확보된다고 판단했다.

### 대안 C: BFS 경로탐색으로 전환

- 외벽 추가에 맞춰 적 AI를 BFS 기반으로 전면 교체.
- 장점: 복잡한 미로 지형에서도 정확한 경로 탐색 가능.
- 기각 이유: 기존 Direct Pursuit + 슬라이딩 방식으로 외벽 우회가 충분히 동작한다. BFS 전환은 스폰 수(최대 180)를 고려할 때 성능 부담이 크고, 기존 AI 동작과의 호환성 문제가 발생한다. ADR-001 결정을 유지한다.

> **후속 변경:** ADR-007에서 이 기각이 번복되었다. 4방향 스폰 도입 후 슬라이딩만으로는 모든 경로를 처리할 수 없게 되어 BFS 거리맵 방식으로 전환했다. 성능 우려는 1회 BFS + 거리맵 캐싱으로 해소되었다.

## 결과

- 맵 크기가 30×24(1440×1152px)로 확대되었다.
- 기지 외벽이 불규칙 타원형 BLOCKED 타일로 변경되었다.
- 입구가 1개(2타일 폭)로 통합되어 WALL 2개로 봉쇄 가능하다.
- 스폰 지점이 맵 하단(row 22)에 5개 균등 배치되었다.
- pursueTarget에 3단계 BLOCKED 슬라이딩, separation에 BLOCKED 롤백 로직이 추가되었다.
- 카메라 초기 위치가 기지 중앙(col 15, row 7 부근)으로 설정되었다.
- NEST_ZONE이 `{13, 16, 3, 5}`로 변경되었다.
- 관련 문서 갱신: `GAME_CONSTANTS.md`, `ENEMY_SYSTEM.md`, `CAMERA_SYSTEM.md`.

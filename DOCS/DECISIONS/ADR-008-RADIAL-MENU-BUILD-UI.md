# ADR-008: 하단 빌드 패널을 Radial Menu 방식으로 교체

> **카테고리:** DECISIONS
> **최초 작성:** 2026-03-23
> **최종 갱신:** 2026-03-23
> **관련 기능:** 건물 배치 UI, 건설 시스템

**상태:** Accepted
**날짜:** 2026-03-23

---

## 맥락

기존 건설 UI는 화면 하단에 고정된 `build-panel`에 6개 건물 버튼(NEST, WALL, THORN, SPORE, REPAIR, RESOURCE)을 나란히 배치하는 방식이었다. 모바일 환경에서 다음 문제가 있었다:

1. **화면 점유:** 하단 72px 높이의 패널이 항상 표시되어 게임 화면을 압축했다.
2. **조작 거리:** 버튼을 누른 후 캔버스의 목표 타일까지 손가락을 이동해야 하므로 터치 동선이 길었다.
3. **건물 수 증가 대비:** 건물 종류가 추가될 경우 버튼이 좁아지거나 스크롤이 필요해지는 확장성 문제가 있었다.

---

## 결정

하단 `build-panel`의 6개 버튼을 제거하고, 중앙 하단에 건설 모드 토글 버튼 1개 + 빈 타일 클릭 시 시계방향 원형 Radial Menu를 표시하는 방식으로 교체한다.

### 구체적 구조

1. **건설 트리거 버튼:** 중앙 하단에 1개 버튼(`build-trigger-btn`)을 고정 배치한다. 클릭 시 건설 모드(buildModeActive)를 토글한다.
2. **Radial Menu:** 건설 모드 활성 상태에서 빈 타일을 클릭하면, 클릭 위치를 중심으로 반경 55px의 원형 메뉴가 DOM 오버레이로 표시된다. 12시 방향부터 시계방향으로 건물 아이콘을 배치한다.
3. **BUILD_GROUPS 배열:** 건물을 그룹으로 관리한다. 현재 1그룹(WALL, THORN, SPORE, REPAIR, RESOURCE)이며, NEST는 PLACING 상태에서 별도 처리한다. 향후 2그룹 분리에 대비한 구조이다.
4. **PLACING 상태:** 건설 트리거 버튼이 숨겨진다. NEST 건설 전에는 일반 건물 배치를 차단한다.

---

## 이유

### 터치 동선 최소화

Radial Menu는 타일을 클릭한 바로 그 위치에 표시되므로, 건물 선택과 배치가 한 동작으로 완료된다. 기존에는 "하단 버튼 클릭 → 캔버스 이동 → 타일 클릭"의 3단계였다.

### 화면 점유 감소

상시 72px 높이의 패널 대신 건설 버튼 1개만 고정 표시된다. Radial Menu는 필요할 때만 잠시 표시되고 건물 선택 즉시 사라진다.

### 확장 가능한 그룹 구조

BUILD_GROUPS 배열로 건물을 그룹화하여, 건물 수가 증가하면 그룹을 나누거나 탭으로 전환하는 확장이 용이하다. 원형 메뉴의 아이콘 수가 6~8개를 넘으면 두 번째 그룹으로 분리할 수 있다.

---

## 검토한 대안

### 대안 A: 기존 빌드 패널 유지 + 스크롤 추가

- 장점: 기존 코드 변경 최소화.
- 기각 이유: 모바일 터치 동선 문제가 해결되지 않는다. 스크롤 패널은 소형 화면에서 조작성이 떨어진다.

### 대안 B: 드래그 앤 드롭 방식

- 하단 패널에서 건물 아이콘을 드래그하여 캔버스에 놓는 방식.
- 장점: 직관적인 조작.
- 기각 이유: 카메라 팬(드래그)과 충돌한다. 드래그와 팬을 구분하는 복잡한 로직이 필요하다.

### 대안 C: Canvas 내부에 메뉴 렌더링

- Radial Menu를 Canvas 2D로 직접 그리는 방식.
- 장점: DOM 의존성 없음.
- 기각 이유: 카메라 변환(zoom, translate) 적용 시 메뉴 크기가 함께 변하며, 클릭 히트 판정에 역변환이 필요하다. DOM 오버레이는 카메라와 독립적으로 동작하여 구현이 단순하다.

---

## 결과

- `build-panel`(하단 6개 버튼)이 제거되었다.
- `build-trigger-bar` + `build-trigger-btn` (중앙 하단 1개 버튼)이 추가되었다.
- `radial-menu` DOM 요소가 `game-container` 내부에 추가되었다.
- `BUILD_GROUPS` 상수 배열이 도입되었다.
- `toggleBuildMode()`, `openRadialMenu()`, `closeRadialMenu()`, `deactivateBuildMode()` 함수가 신설되었다.
- `BUILD_PANEL_H` 레이아웃 상수는 유지되나, 실질적으로 build-trigger-bar의 높이(버튼 1개)로 축소되었다.
- 기존 `buildBuildPanel()`, `updateBuildPanel()` 함수의 역할이 변경되었다.

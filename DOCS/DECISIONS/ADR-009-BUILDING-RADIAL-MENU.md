# ADR-009: 건물 클릭 시 정보 패널 대신 Radial Menu 표시

> **카테고리:** DECISIONS
> **최초 작성:** 2026-03-23
> **최종 갱신:** 2026-03-23
> **관련 기능:** 건물 선택 UI, 건물 정보 패널, 건물 업그레이드/철거

**상태:** Accepted
**날짜:** 2026-03-23

---

## 맥락

기존에는 건물을 클릭하면 `openBuildingPanel()`이 직접 호출되어 DOM 기반 정보 패널이 열렸다. 패널 안에 업그레이드 버튼, 철거 버튼, 상세 스탯이 모두 포함되어 있었다. 모바일 환경에서 다음 문제가 있었다:

1. **조작 단계 과다:** 건물 업그레이드를 하려면 건물 클릭 → 패널 내 업그레이드 버튼 클릭의 2단계가 필요했다. 패널이 화면 하단에 표시되므로 시선 이동도 발생했다.
2. **정보 과부하:** 단순히 업그레이드만 하고 싶은데 전체 스탯 정보가 한꺼번에 표시되어 시각적으로 복잡했다.
3. **건설 Radial Menu와의 일관성 부재:** ADR-008에서 건설 UI를 Radial Menu로 교체했으나, 건물 관리 UI는 여전히 패널 방식이어서 조작 패턴이 불일치했다.

---

## 결정

건물 클릭 시 `openBuildingPanel()` 대신 `openBuildingRadialMenu(clientX, clientY, building)`를 호출하여 원형 액션 메뉴를 먼저 표시한다. 메뉴에서 "정보" 아이콘을 선택해야 기존 정보 패널이 열린다.

### 메뉴 구성

- **NEST:** 진화(상향 화살표) + 정보(i) = 2개 액션. NEST는 철거 불가이므로 철거 아이콘을 생략한다.
- **일반 건물:** 진화 + 철거(휴지통) + 정보 = 3개 액션.
- **중앙 표시:** 건물명 + HP + 레벨.

### 진화 액션의 즉시 실행

진화 아이콘 클릭 시 `startUpgrade(building)`을 즉시 호출하고, 성공하면 정보 패널을 열어 결과를 확인할 수 있게 한다. 실패 시 상태 메시지를 표시한다. 이를 통해 업그레이드가 1클릭(건물 클릭 → 진화 아이콘)으로 완료된다.

---

## 이유

### 빈번한 액션의 터치 동선 최소화

업그레이드는 게임 중 가장 빈번한 건물 조작이다. 건물 위치에 Radial Menu가 뜨고 진화 아이콘을 바로 누를 수 있으므로, 시선 이동과 손가락 이동이 최소화된다. 기존 패널 방식에서는 화면 하단까지 시선과 손가락을 이동해야 했다.

### 정보 요청은 명시적으로

상세 스탯 확인은 업그레이드/철거보다 빈도가 낮다. 정보를 별도 아이콘으로 분리하여, 필요할 때만 접근하게 한다. 대부분의 조작에서 패널이 열리지 않아 화면 점유가 줄어든다.

### ADR-008과의 UI 일관성

건설(빈 타일 클릭 → Radial Menu)과 건물 관리(건물 클릭 → Radial Menu) 모두 원형 메뉴 패턴으로 통일된다.

---

## 검토한 대안

### 대안 A: 기존 패널 유지 + 상단에 빠른 버튼 추가

- 패널 상단에 업그레이드/철거 버튼을 크게 배치하고, 하단에 상세 정보를 축소 표시.
- 기각 이유: 패널 자체의 화면 점유 문제가 해결되지 않는다. 건물 위치에서 패널까지의 시선 이동은 여전히 발생한다.

### 대안 B: 길게 누르기(long press)로 패널, 짧게 누르기로 즉시 업그레이드

- 기각 이유: 카메라 팬(드래그)과 길게 누르기의 구분이 어렵다. 실수로 업그레이드가 실행될 위험이 있다.

---

## 결과

- `openBuildingRadialMenu(clientX, clientY, building)` 함수가 신설되었다.
- `onTileClicked()` 내 건물 클릭 분기에서 `openBuildingPanel` 대신 `openBuildingRadialMenu`를 호출한다.
- 기존 `openBuildingPanel()`은 Radial Menu의 "정보" 액션에서만 호출된다.
- 건물 업그레이드가 Radial Menu에서 1클릭으로 가능해졌다.
